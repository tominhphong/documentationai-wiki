## Giới thiệu

ClaudeKit Engineer là plugin kỹ thuật toàn diện nhất trong bộ ClaudeKit — tập hợp khoảng 75 skills chuyên biệt cho development, debugging, deployment, và operations.

Mỗi skill được kích hoạt bằng slash command `/<skill-name>` trong Claude Code, hoặc tự động thông qua Skill tool khi agent nhận diện task phù hợp. Trước mỗi task, dùng skill `find-skills` để discover các skills liên quan thay vì improvise:

```
/find-skills <query>
```

Ví dụ: `/find-skills database migration` sẽ surface `databases`, `deploy`, và các skills liên quan database.

## 1. Planning and Research

Nhóm skills hỗ trợ lên kế hoạch, nghiên cứu, và phân tích trước khi bắt tay vào code.

#### **ck-plan**

- **Purpose:** Tạo implementation plan chi tiết cho feature hoặc project, bao gồm phases, tasks, và dependencies.
- **When to use:** Trước mọi feature mới — đặc biệt khi scope không rõ ràng hoặc có nhiều files cần thay đổi.
- **Example command:** `/ck-plan implement user authentication with JWT`

#### **brainstorm**

- **Purpose:** Generate ý tưởng sáng tạo và đa dạng cho một vấn đề, không giới hạn bởi constraints.
- **When to use:** Khi cần explore options trước khi commit vào một hướng cụ thể.
- **Example command:** `/brainstorm ways to improve API response caching`

#### **research**

- **Purpose:** Nghiên cứu sâu về một chủ đề kỹ thuật, tổng hợp thông tin từ nhiều nguồn.
- **When to use:** Cần background knowledge về library, pattern, hoặc technology mới trước khi implement.
- **Example command:** `/research best practices for React Server Components`

#### **scout**

- **Purpose:** Khảo sát nhanh codebase hoặc repo để hiểu structure, patterns, và conventions đang dùng.
- **When to use:** Mở đầu session mới trên codebase chưa quen, hoặc cần nắm context nhanh.
- **Example command:** `/scout` — tự động scan codebase hiện tại

#### **sequential-thinking**

- **Purpose:** Phân tích vấn đề phức tạp theo từng bước logic, tránh nhảy vọt đến kết luận.
- **When to use:** Bug khó reproduce, architecture decision có nhiều trade-offs, hoặc debugging session bế tắc.
- **Example command:** `/sequential-thinking why does the auth token expire prematurely`

#### **find-skills**

- **Purpose:** Discover skills phù hợp với task hiện tại từ toàn bộ catalog ClaudeKit.
- **When to use:** Đầu mỗi task — check xem đã có skill sẵn trước khi tự làm.
- **Example command:** `/find-skills seo content writing`

#### **context-engineering**

- **Purpose:** Tối ưu hóa context window — cắt bỏ noise, giữ lại thông tin quan trọng, cải thiện quality của AI response.
- **When to use:** Session dài bắt đầu chậm, hoặc khi muốn cải thiện prompt quality cho một task phức tạp.
- **Example command:** `/context-engineering` — analyze và suggest improvements

#### **ask**

- **Purpose:** Đặt câu hỏi làm rõ yêu cầu trước khi thực thi, tránh build sai.
- **When to use:** Task description mơ hồ hoặc có nhiều cách hiểu hợp lý.
- **Example command:** `/ask` — generate clarifying questions cho task hiện tại

#### **problem-solving**

- **Purpose:** Framework giải quyết vấn đề có cấu trúc — define problem, root cause analysis, solution options.
- **When to use:** Gặp technical blocker mà sequential-thinking chưa đủ, hoặc cần present solution options rõ ràng.
- **Example command:** `/problem-solving database connection pool exhaustion in production`

## 2. Coding Core

Nhóm skills cốt lõi cho việc viết và phát triển code.

#### **cook**

- **Purpose:** Activate trước mỗi lần implement feature mới — ensures code follows best practices, patterns, và conventions của codebase.
- **When to use:** Kích hoạt TRƯỚC khi bắt đầu implement bất kỳ feature nào. Đây là skill bắt buộc trước `frontend-development`, `backend-development`, hoặc `mobile-development`.
- **Example command:** `/cook` — run trước khi bắt đầu coding session

#### **fix**

- **Purpose:** Activate trước khi sửa bugs — phân tích root cause, đề xuất fix minimal, tránh over-engineering.
- **When to use:** Kích hoạt TRƯỚC khi sửa bất kỳ bug nào. Tránh fix triệu chứng thay vì nguyên nhân.
- **Example command:** `/fix` — run khi nhận bug report hoặc failing test

#### **frontend-development**

- **Purpose:** Best practices cho frontend — component structure, state management, performance, accessibility.
- **When to use:** Implement UI components, pages, hoặc client-side logic.
- **Example command:** `/frontend-development build a responsive product card grid`

#### **backend-development**

- **Purpose:** Best practices cho backend — API design, error handling, database patterns, security.
- **When to use:** Build API endpoints, services, middleware, hoặc business logic.
- **Example command:** `/backend-development create REST API for user profile`

#### **mobile-development**

- **Purpose:** Patterns và best practices cho mobile development — iOS/Android native hoặc cross-platform.
- **When to use:** Build mobile screens, handle platform-specific behaviors, optimize for mobile constraints.
- **Example command:** `/mobile-development implement push notification handler`

#### **web-frameworks**

- **Purpose:** Reference cho các web frameworks phổ biến — Next.js, Remix, SvelteKit, Astro, và routing patterns.
- **When to use:** Cần framework-specific conventions, migration guidance, hoặc framework feature usage.
- **Example command:** `/web-frameworks Next.js app router data fetching patterns`

#### **test**

- **Purpose:** Viết unit tests và integration tests cho code hiện tại, đảm bảo coverage đủ cao.
- **When to use:** Sau khi implement feature mới, hoặc khi cần add test coverage cho legacy code.

#### **web-testing**

- **Purpose:** E2E testing cho web applications — Playwright, Cypress, hoặc testing framework phù hợp.
- **When to use:** Cần test critical user flows end-to-end, form submissions, hoặc navigation flows.
- **Example command:** `/web-testing checkout flow from cart to confirmation`

## 4. Review and Quality

Nhóm skills đảm bảo code quality, security, và maintainability.

#### **code-review**

- **Purpose:** Review code về quality, readability, patterns, và potential issues — như senior engineer review.
- **When to use:** Sau khi implement xong, trước khi commit hoặc tạo PR.
- **Example command:** `/code-review` — review diff hoặc file được chỉ định

#### **security-scan**

- **Purpose:** Scan code tìm security vulnerabilities — OWASP Top 10, injection, auth flaws, hardcoded secrets.
- **When to use:** Trước khi deploy hoặc khi code liên quan authentication, authorization, user data, payments.
- **Example command:** `/security-scan` — scan codebase hoặc specific files

#### **ck-security**

- **Purpose:** Security audit sâu hơn security-scan — kiến trúc security, threat modeling, compliance checks.
- **When to use:** Major feature release, security audit định kỳ, hoặc khi architecture thay đổi lớn.
- **Example command:** `/ck-security audit authentication flow`

## 5. Debugging

Nhóm skills chuyên biệt cho debugging và troubleshooting.

#### **ck-debug**

- **Purpose:** Systematic debugging — reproduce, isolate, root cause analysis, fix, verify.
- **When to use:** Bug không rõ nguyên nhân, sau khi `/fix` không đủ context.
- **Example command:** `/ck-debug users getting logged out randomly`

#### **ck-predict**

- **Purpose:** Dự đoán potential bugs và edge cases trước khi chúng xảy ra trong production.
- **When to use:** Sau khi implement feature phức tạp, trước khi deploy lên production.
- **Example command:** `/ck-predict` — analyze current changes for risk areas

#### **ck-scenario**

- **Purpose:** Generate test scenarios và edge cases cho một feature, đảm bảo không bỏ sót cases quan trọng.
- **When to use:** Khi viết test plan, hoặc cần validate feature handling của unusual inputs.
- **Example command:** `/ck-scenario file upload with various file types and sizes`

#### **ck-autoresearch**

- **Purpose:** Tự động research giải pháp cho error message hoặc bug description, tổng hợp từ docs và community.
- **When to use:** Gặp error message lạ, dependency conflict, hoặc framework behavior không hiểu.
- **Example command:** `/ck-autoresearch "TypeError: Cannot read properties of undefined (reading 'map')"`

## 6. Build and Deploy

Nhóm skills cho build pipeline, deployment, và git operations.

#### **deploy**

- **Purpose:** Deploy application lên hosting platforms — Vercel, Netlify, Cloudflare Workers, hoặc custom servers.
- **When to use:** Khi cần deploy lên staging hoặc production, hoặc setup deployment pipeline mới.
- **Example command:** `/deploy to Vercel with environment variables`

#### **devops**

- **Purpose:** Infrastructure, CI/CD pipelines, Docker, Kubernetes, và cloud configurations.
- **When to use:** Setup CI/CD, Dockerize app, cấu hình cloud resources, hoặc troubleshoot infrastructure issues.
- **Example command:** `/devops setup GitHub Actions CI for Node.js app`

#### **git**

- **Purpose:** Conventional commits, branch strategies, safe git operations, PR workflow.
- **When to use:** Mọi git operation — commit, branch, merge, rebase. Đặc biệt dùng trước khi push.
- **Example command:** `/git` — generate commit message từ current diff

#### **worktree**

- **Purpose:** Quản lý git worktrees cho parallel development — setup, switch, cleanup worktrees.
- **When to use:** Cần work trên nhiều branches cùng lúc, hoặc setup Agent Team với isolated worktrees.
- **Example command:** `/worktree setup feature/new-auth in parallel worktree`

#### **ship**

- **Purpose:** Full shipping pipeline — merge main, test, code review, commit, push. Orchestrates toàn bộ quy trình.
- **When to use:** Khi feature hoàn chỉnh và ready to ship — skill này chạy end-to-end.
- **Example command:** `/ship` — trigger full pre-ship checklist và deploy pipeline

## Discovery Flow

Để tận dụng tối đa catalog 75 skills:

**Bước 1 — Find skills trước mọi task:**

```
/find-skills <domain hoặc task description>
```

**Bước 2 — Chain skills phù hợp:**

- Build feature: `/cook` → implement → `/test` → `/code-review` → `/git` → `/ship`
- Fix bug: `/fix` → `/ck-debug` → implement fix → `/test` → `/git`
- New project: `/ck-plan` → `/bootstrap` → `/cook` → build phases

**Tài liệu liên quan:**

- [Skill Playbooks](/workflows/skill-playbooks) — pre-composed chains cho common use cases
- [Quick Reference](/knowledge-qa/skill-quick-reference) — lookup table toàn bộ skills
