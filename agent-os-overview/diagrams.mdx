**Key concepts:**
- Brain (Opus) is the sole orchestrator — never a worker
- Workers commit to separate git worktrees to avoid merge conflicts
- All coordination is filesystem-based: no runtime IPC, no shared memory
- Win 4 acts as QA gate before Brain merges to main

**Why it matters:** Parallel workers multiply throughput. A 5-agent sprint completes in the time of 1 worker because each agent operates on a non-overlapping file scope.

**Key concepts:**
- Palace → Wings → Rooms → Drawers → Items: 5-level hierarchy
- Tunnels cross-link items between Wings (no duplication needed)

**Why it matters:** Structured hierarchy prevents knowledge fragmentation. Agent finds the same fact in one place every time — DRY principle applied to memory.

- Win 4 (QA/reference) runs Haiku; Win 2/3 (content authors) run Sonnet

**Why it matters:** Routing to the wrong model is either wasteful (Opus for trivial tasks) or risky (Haiku for decisions that shape the entire system). Cost-aware routing keeps the system economically sustainable.
