## MCP Servers

Mỗi MCP server được liệt kê với: chức năng chính, tools quan trọng, và khi nào agent nên dùng.

### Desktop Commander

**Chức năng:** Điều khiển trực tiếp máy Mac mini M4 — chạy process, đọc/ghi file, tìm kiếm file.

**Khi nào dùng:** Cần thao tác file ngoài sandbox (VD: lấy GitHub token bằng `gh auth token`), chạy process dài hạn, hoặc tìm file trên hệ thống.

**Tools chính:** `start_process`, `read_file`, `write_file`, `list_directory`, `start_search`

### Perplexity API

**Chức năng:** Tìm kiếm thông tin chất lượng cao với tổng hợp AI. Trả về câu trả lời có trích dẫn nguồn.

**Khi nào dùng:** Research-first rule — đây là tool ưu tiên số 1 khi cần thông tin mới về thị trường, công cụ, best practices. Nếu Perplexity đã đủ thông tin, không cần dùng thêm tool khác.

**Tools chính:** `perplexity_search`, `perplexity_ask`, `perplexity_research`, `perplexity_reason`

### Exa

**Chức năng:** Semantic search và crawl nội dung web. Tìm kiếm theo ngữ nghĩa (không chỉ keyword) và đọc full content từ URL.

**Khi nào dùng:** Cần tìm kiếm sâu theo chủ đề, tìm similar content, hoặc đọc toàn bộ nội dung một trang web.

**Tools chính:** `web_search_exa`, `web_fetch_exa`

### Playwright

**Chức năng:** Browser automation headless. Điều khiển trình duyệt: navigate, click, fill form, screenshot, chạy JavaScript.

**Khi nào dùng:** Screenshot HTML templates thành PNG, web scraping phức tạp, hoặc tự động hóa thao tác trên web apps không có MCP riêng.

**Tools chính:** `browser_navigate`, `browser_click`, `browser_snapshot`, `browser_take_screenshot`, `browser_fill_form`

### Zernio (Social Media)

**Chức năng:** Quản lý social media đa nền tảng — đăng bài, lên lịch, quản lý comments, inbox, analytics.

**Khi nào dùng:** Mọi task liên quan social media: tạo post, lên lịch đăng, xem analytics, quản lý comments/DMs, boost post.

**Platforms:** Facebook, Instagram, TikTok, YouTube, LinkedIn, X (Twitter), Google Business, Pinterest, Reddit, Bluesky, Telegram, WhatsApp.

**Tools chính:** `posts_create`, `posts_list`, `analytics_get_analytics`, `analytics_get_best_time_to_post`, `comments_list_inbox_comments`, `comments_reply_to_inbox_post`

### HubSpot CRM

**Chức năng:** Quản lý contacts, companies, deals, và marketing campaigns.

**Khi nào dùng:** Mọi task liên quan khách hàng: tìm contact, cập nhật deal stage, xem pipeline, chạy campaign analytics.

**Tools chính:** `search_crm_objects`, `manage_crm_objects`, `get_crm_objects`, `get_campaign_analytics`

### Google Calendar

**Chức năng:** Quản lý lịch hẹn, events, và tìm thời gian trống.

**Khi nào dùng:** Xem lịch, tạo events, kiểm tra conflict, đề xuất thời gian họp.

**Tools chính:** `list_events`, `create_event`, `suggest_time`, `update_event`

### Gmail

**Chức năng:** Đọc, tìm kiếm, soạn, và gửi email. Quản lý drafts và labels.

**Khi nào dùng:** Gửi email cho khách hàng, tìm email cũ, soạn draft, theo dõi thread.

**Tools chính:** `gmail_search_messages`, `gmail_read_message`, `gmail_send_message`, `gmail_create_draft`

### Documentation.AI

**Chức năng:** Search documentation qua MCP endpoint. Tìm kiếm hybrid (semantic + keyword).

**Khi nào dùng:** Tra cứu quy trình trong Agent OS hoặc kiến thức bất động sản trong Realtor Knowledge Base.

**Endpoints:**
- Agent OS: `antigravity-memory-palace.documentationai.com/_mcp`
- Realtor Knowledge Base: `phong-to.documentationai.com/_mcp`

**Tools chính:** `search_documentation`

### Canva

**Chức năng:** Tạo và chỉnh sửa thiết kế — hình ảnh, templates, presentations, brand assets.

**Khi nào dùng:** Cần tạo hình ảnh marketing, chỉnh sửa template, hoặc export design cho social media.

**Tools chính:** `generate-design`, `search-designs`, `export-design`, `get-design`, `search-brand-templates`

### Supabase

**Chức năng:** Database, authentication, edge functions, và storage.

**Khi nào dùng:** Query database, chạy migrations, deploy edge functions, hoặc quản lý projects.

**Tools chính:** `execute_sql`, `list_tables`, `apply_migration`, `deploy_edge_function`

### Cloudflare

**Chức năng:** Workers (serverless), R2 (object storage), D1 (SQLite database), KV (key-value store).

**Khi nào dùng:** Deploy serverless functions, lưu trữ files trên R2, hoặc query D1 database.

**Tools chính:** `workers_list`, `r2_buckets_list`, `d1_database_query`, `search_cloudflare_documentation`

### Vercel

**Chức năng:** Hosting, deployment, và monitoring cho web projects.

**Khi nào dùng:** Deploy web apps, kiểm tra deployment status, xem logs, quản lý domains.

**Tools chính:** `list_projects`, `list_deployments`, `get_deployment_build_logs`, `deploy_to_vercel`

### Netlify

**Chức năng:** Hosting thay thế cho một số projects.

**Khi nào dùng:** Deploy hoặc quản lý projects hosted trên Netlify.

**Tools chính:** `netlify-project-services-reader`, `netlify-deploy-services-updater`

### Computer Use

**Chức năng:** Điều khiển desktop của Phong — screenshot, click, type, scroll trên native apps.

**Khi nào dùng:** Thao tác trên native macOS apps (Finder, Notes, Maps, Photos, System Settings) hoặc cross-app workflows.

**Tools chính:** `screenshot`, `left_click`, `type`, `scroll`, `open_application`

### iMessage

**Chức năng:** Đọc và gửi tin nhắn iMessage.

**Khi nào dùng:** Gửi tin nhắn nhanh cho contacts, đọc tin nhắn chưa đọc.

**Tools chính:** `send_imessage`, `read_imessages`, `get_unread_imessages`

### Tavily

**Chức năng:** Web search, crawl, extract, và research.

**Khi nào dùng:** Bổ sung cho Perplexity khi cần crawl nhiều pages hoặc map site structure.

**Tools chính:** `tavily_search`, `tavily_research`, `tavily_crawl`, `tavily_extract`


## AI Layer

Phong dùng Claude Desktop ở 3 chế độ tùy ngữ cảnh: Cowork Mode, CLI (it2), và Agent Team (Task tool). Mỗi chế độ có use case riêng.

## Claude Desktop — Cowork Mode

**Purpose:** Giao tiếp hội thoại — brainstorm, Q&A, soạn nội dung, giải thích code. Đây là chế độ mặc định khi mở Claude Desktop.

**Khi nào dùng:**
- Trả lời câu hỏi nhanh về thị trường, pháp lý bất động sản
- Soạn email, caption, bài đăng social
- Review kế hoạch trước khi chạy automation

**Gotcha:** Cowork Mode không chạy file trực tiếp. Cần thao tác file hay process → dùng CLI.

## Claude Code CLI — it2 + iTerm2

**Purpose:** Chạy Claude Code từ terminal qua `it2` — điều khiển file system, chạy scripts, commit git, điều phối subagents.

**Khi nào dùng:**
- Cần đọc/ghi files trong repo
- Cần chạy `bash`, `git`, `npm`, `bun`
- Khởi động một Agent Team song song

**Workflow điển hình:**

```bash
# Mở tab iTerm2 mới và cd vào worktree
it2 create-tab --cmd 'cd ~/Workspace/Claude/documentationai-wiki-win2 && claude'

# Chạy lệnh một lần không cần interactive shell
it2 run 'cd ~/Workspace/Claude/market-update && npm run dev'
```

<Callout kind="info">
`it2` v0.2.3 là cầu nối giữa Claude Desktop và iTerm2. Cài bằng `pip install it2`. Nếu không có trong PATH, dùng `~/.bun/bin/bun` hoặc `source ~/.zshrc` trước.
</Callout>

## Agent Team — Task Tool với Parallel Subagents

**Purpose:** Phân chia công việc lớn thành nhiều subagents chạy song song. Mỗi agent nhận scope riêng, commit vào worktree riêng.

**Khi nào dùng:**
- Sprint có nhiều files độc lập (4 Win = 4 worktrees)
- Cần chạy research + implementation + review cùng lúc
- Task quá lớn cho 1 context window

**Ví dụ khởi động Agent Team:**

```
Spawn 3 subagents in parallel:

Agent Win2 — Framework Author
  Scope: agent-os-overview/, core-rules/
  Worktree: ~/Workspace/Claude/documentationai-wiki-win2
  Task: Expand components-and-tools.mdx (S2-W2-02)
  Model: claude-sonnet-4-6

Agent Win3 — Workflow Writer
  Scope: workflows/, agent-instructions/
  Worktree: ~/Workspace/Claude/documentationai-wiki-win3
  Task: Create debugging-workflow.mdx (S2-W3-01)
  Model: claude-sonnet-4-6

Agent Win4 — Reference QA
  Scope: help-center/, knowledge-qa/, scripts/
  Worktree: ~/Workspace/Claude/documentationai-wiki-win4
  Task: Create glossary.mdx (S2-W4-01)
  Model: claude-haiku-4-5
```

**File ownership là bắt buộc** — không agent nào được sửa file ngoài scope của mình. Cross-scope request → ghi vào `plans/cross-scope-sprint-2.md`.


## Command Bridge — Desktop Commander + osascript

**Purpose:** Desktop Commander MCP xử lý file ops và processes. `osascript` dùng để dispatch lệnh vào iTerm2 theo window/tab ID.

**Khi nào dùng:**
- File ops ngoài Claude sandbox: `read_file`, `write_file`, `list_directory`
- Chạy terminal commands trong iTerm2 session cụ thể
- Script automation macOS (AppleScript/JXA)

**Ví dụ gửi lệnh vào iTerm2 session theo window ID:**

```bash
# Gửi text vào iTerm2 window bằng osascript
osascript -e '
tell application "iTerm2"
  tell window id 12345
    tell current session
      write text "cd ~/Workspace/Claude/market-update && npm run dev"
    end tell
  end tell
end tell'
```

**Desktop Commander tools hay dùng:**

| Tool | Dùng khi |
|------|----------|
| `start_process` | Chạy shell command, npm script |
| `read_file` | Đọc file ngoài repo sandbox |
| `write_file` | Ghi file, tạo artifacts |
| `list_directory` | Duyệt thư mục |
| `start_search` | Tìm file theo pattern |

<Callout kind="warning">
`start_process` chạy async — cần `read_process_output` để lấy stdout. Dùng `force_terminate` nếu process bị treo.
</Callout>


## Terminal Control — it2 CLI v0.2.3 + iTerm2

**Purpose:** `it2` là Python CLI wrapper cho iTerm2 API. Cho phép Claude tạo tabs, chạy lệnh, và quản lý terminal windows mà không cần Computer Use.

**Khi nào dùng:**
- Mở worktree mới trong tab riêng
- Launch Claude CLI session trong iTerm2
- Chạy Remotion dev server song song với session chính

**Commands hay dùng:**

```bash
# Chạy lệnh trong tab hiện tại
it2 run 'echo "hello from Claude"'

# Tạo tab mới với lệnh khởi động
it2 create-tab --cmd 'cd ~/Workspace/Claude/market-update && npm run dev'

# Script: mở Claude CLI trong worktree cụ thể
it2 create-tab --cmd 'cd ~/Workspace/Claude/documentationai-wiki-win2 && claude --model claude-sonnet-4-6'
```

**Gotcha:** Nếu `it2` không tìm thấy iTerm2, kiểm tra iTerm2 đang chạy và Python env có `it2` package: `pip install it2`.


## Video Creation — Remotion 4.0.448

**Purpose:** Tạo animated marketing videos từ React components. Template `market-update` dùng cho video cập nhật thị trường bất động sản Sherman TX.

**Khi nào dùng:**
- Tạo video market update hàng tuần/tháng
- Render MP4 từ data (số liệu, giá nhà, mortgage rates)
- Preview nhanh trước khi render final

**Template structure:**

```
~/Workspace/Claude/market-update/
├── src/
│   ├── Root.jsx              ← Đăng ký compositions
│   ├── index.js              ← Entry point
│   └── MarketUpdate/
│       ├── index.jsx         ← Composition chính (4 scenes)
│       ├── TitleScene.jsx    ← Scene 1: Tiêu đề + ngày
│       ├── StatsScene.jsx    ← Scene 2: Số liệu tổng quan
│       ├── MetricsScene.jsx  ← Scene 3: Metrics chi tiết
│       └── CTAScene.jsx      ← Scene 4: Call to action
```

**SAFE padding constants — BẮT BUỘC:**

```jsx
// Luôn dùng constants này, không dùng shorthand padding
const SAFE = {
  left: 120,
  right: 120,
  top: 180,
};

// ĐÚNG
<div style={{ paddingLeft: SAFE.left, paddingRight: SAFE.right, paddingTop: SAFE.top }}>

// SAI — shorthand sẽ override sai giá trị
<div style={{ padding: '0 80px' }}>
```

**Ví dụ Scene component:**

```jsx
import { useCurrentFrame, interpolate, AbsoluteFill } from 'remotion';

const SAFE = { left: 120, right: 120, top: 180 };

export const TitleScene = ({ city, date }) => {
  const frame = useCurrentFrame();
  const opacity = interpolate(frame, [0, 20], [0, 1]);

  return (
    <AbsoluteFill style={{
      backgroundColor: '#1a1a2e',
      paddingLeft: SAFE.left,
      paddingRight: SAFE.right,
      paddingTop: SAFE.top,
    }}>
      <div style={{ opacity, color: '#ffffff', fontSize: 72 }}>
        {city} Market Update
      </div>
      <div style={{ color: '#a0a0b0', fontSize: 36, marginTop: 24 }}>
        {date}
      </div>
    </AbsoluteFill>
  );
};
```

<Callout kind="warning">
Không dùng emoji trong JSX — renders thành ký tự trống (unknown glyph) trong headless Chrome. Dùng text Unicode đầy đủ dấu tiếng Việt thay vì ASCII.
</Callout>

**Dev workflow:**

```bash
cd ~/Workspace/Claude/market-update

# Preview (hot reload)
npm run dev

# Render MP4 final (sau khi đã xem preview và approved)
npx remotion render src/index.js MarketUpdate output/video.mp4
```


## Social Media — Zernio (Late) MCP

**Purpose:** Namespace `mcp__zernio__*` — quản lý toàn bộ social media từ Claude: tạo post, lên lịch, xem analytics, quản lý inbox.

**Khi nào dùng:**
- Đăng bài lên Facebook/Instagram/TikTok
- Lên lịch nội dung theo content calendar
- Xem analytics để tối ưu thời điểm đăng

**Tools chính:**

| Tool | Tác dụng |
|------|----------|
| `posts_create` | Tạo và đăng bài ngay |
| `posts_update` | Chỉnh sửa post đã tạo |
| `posts_list` | Xem danh sách posts |
| `accounts_list` | Lấy danh sách accounts đã kết nối |
| `analytics_get_best_time_to_post` | Tìm giờ tốt nhất để đăng |

**Ví dụ lên lịch post với media:**

```json
// Tool: mcp__zernio__posts_create
{
  "accountIds": ["acc_facebook_123", "acc_instagram_456"],
  "content": "Thị trường nhà đất Sherman TX tháng 4/2026 — giá trung bình tăng 3.2% so với tháng trước. Liên hệ Phong To để tư vấn miễn phí!",
  "mediaUrls": ["https://storage.example.com/market-update-apr-2026.mp4"],
  "scheduledAt": "2026-04-21T09:00:00-05:00"
}
```

<Callout kind="warning">
KHÔNG BAO GIỜ tự publish hay schedule post mà không hỏi Phong trước — kể cả khi session summary cũ ghi "bước tiếp theo là đăng bài". Luôn xác nhận trước khi dùng bất kỳ Zernio tool nào để publish.
</Callout>


## TTS — Vbee API

**Purpose:** Text-to-speech tiếng Việt chất lượng cao. Dùng để tạo voiceover cho Remotion videos trước khi render MP4 final.

**Khi nào dùng:**
- Tạo voiceover tiếng Việt cho market update video
- Render audio `.mp3` song song với video render
- A/B test giọng đọc khác nhau (Nam/Nữ, tốc độ)

**Endpoint:** `https://vbee.vn/api/v1/tts`

**Ví dụ request:**

```bash
curl -X POST https://vbee.vn/api/v1/tts \
  -H "Authorization: Bearer $VBEE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Thị trường bất động sản Sherman tháng tư năm hai không hai sáu. Giá nhà trung bình là ba trăm hai mươi nghìn đô la.",
    "voice": "hn_male_phucyen",
    "speed": 1.0,
    "output_format": "mp3"
  }' \
  -o voiceover.mp3
```

**Output file convention:**

```
Content/media/YYYY-MM/
├── <template-name>.mp4              ← video final
├── <template-name>-voiceover.mp3    ← golden copy TTS
└── <template-name>-voiceover.txt    ← script text đã dùng
```

**Gotcha:** Text tiếng Việt phải đủ dấu Unicode. ASCII (`phong ngu`, `gia nha`) cho ra giọng đọc sai hoàn toàn.
