## Scenario Setup

**Goal:** Implement `POST /api/submit-lead` Cloudflare Worker endpoint with request validation, tests, and endpoint reference docs тАФ in one parallel sprint.

### Agent Roster

| Agent | Role | Model | File Ownership |
|-------|------|-------|----------------|
| Lead | Orchestrator + merge | Opus | `plans/*`, `wrangler.toml`, `src/index.ts` (merge only) |
| Dev | Endpoint implementation | Sonnet | `src/workers/submit-lead.ts`, `src/types/*.ts` |
| Tester | Test suite | Sonnet | `tests/submit-lead.test.ts`, `tests/fixtures/*` |
| Docs | Reference + changelog | Haiku | `docs/api/submit-lead.md`, `docs/changelog.md` |

**No overlap** тАФ each glob is distinct. Dev never touches `tests/`. Tester never edits `src/workers/`.

### Prerequisites

- Team config at `~/.claude/teams/wiki-team/config.json` listing all four agents by name
- Git worktrees: Dev in `worktree-dev/`, Tester in `worktree-tester/`, Docs in `worktree-docs/`
- Tasks pre-created by Lead: T-1 (endpoint), T-2 (tests), T-3 (docs)

## Data Handoff Points

Three concrete handoffs with format and content.

### Handoff A тАФ Dev to Tester (endpoint contract)

**When:** Dev completes T-1, before Tester claims T-2.

**What Dev sends:**
- File path of the endpoint implementation
- Route + HTTP method
- Request body schema (field names, types, required vs optional)
- Response shape for success and each error case
- Explicit statement: "no test coverage yet"

**Tester reads and starts:** `tests/submit-lead.test.ts` тАФ covering valid payload, each required-field-missing case, optional fields, and edge cases.

---

### Handoff B тАФ Tester to Docs (test results + edge cases)

**When:** Tester completes T-2, before Docs claims T-3.

**What Tester sends:**
- Test file path
- Total tests + pass/fail count
- List of edge cases discovered during testing (not in original spec)
- Any bugs found and whether they were fixed in the source

**Docs reads and writes:** endpoint reference, marking edge cases with `<Note>` callouts.

---

### Handoff C тАФ All workers to Lead (completion signals)

**When:** Each worker finishes their task (can arrive at different times).

**What each worker sends:**
- Task ID completed
- Primary output (file path or brief description)
- Any caveats Lead needs for merge (e.g., "T-1 was revised once after T-2 found a bug")

**Lead buffers** until all 3 signals received, then runs merge sequence.

## Failure Branch тАФ Tester Finds Bug Mid-Sprint

This is the most common failure scenario. Walk-through:

**Step 1 тАФ Tester detects failure:**

Tester writes test for empty-string email тЖТ test fails тЖТ endpoint returns 200 when it should return 400. This is a Dev scope bug (src/workers/ is Dev's glob).

**Step 2 тАФ Tester reports to Dev (not Lead):**

```
Tester тЖТ Dev: "Bug in T-1 тАФ empty string email returns 200. Repro at tests/submit-lead.test.ts:34. Need fix before I can complete T-2."
```

Tester does NOT edit `src/workers/` тАФ that violates file ownership. Tester pauses on T-2 while waiting.

**Step 3 тАФ Dev re-opens and fixes:**

```
TaskUpdate("T-1", "in_progress")   // re-open
```

Dev applies fix, pushes to worktree branch.

```
TaskUpdate("T-1", "completed")     // close again
SendMessage to Tester: "Fix pushed тАФ pull and re-verify."
```

**Step 4 тАФ Tester re-verifies:**

Tester pulls Dev's branch, runs tests again. If pass тЖТ continues T-2. If still failing тЖТ repeats Step 2.

**Lead is NOT involved** in this loop unless Dev cannot fix after 2 attempts, in which case Tester escalates:

```
Tester тЖТ Lead: "T-1 bug not resolved after 2 attempts. T-2 is blocked. Needs lead intervention."
TaskUpdate("T-2", "blocked")
```

## Completion + Merge

**Lead receives three completion signals** (they may arrive out of order тАФ that is fine).

**Lead merge sequence:**

```bash
# In Lead's main worktree
git fetch --all

# Merge Dev branch first (source of truth for implementation)
git merge worktree-dev --no-ff -m "feat: add POST /api/submit-lead endpoint"

# Merge Tester branch (tests reference the implementation)
git merge worktree-tester --no-ff -m "test: add submit-lead test suite (12 tests)"

# Merge Docs branch last (references both implementation and test results)
git merge worktree-docs --no-ff -m "docs: add submit-lead endpoint reference"

# Deploy
wrangler deploy
```

**If merge conflict:** Lead resolves directly тАФ never asks a worker to edit outside their scope at merge time.

**After deploy:** Lead broadcasts:

```
Lead тЖТ broadcast: "Sprint complete. POST /api/submit-lead live. Merge: dev + tester + docs branches all merged to main. Worktrees can be cleaned up."
```
