## Overview

Hướng dẫn này dành cho người mới cần setup **documentationai-wiki** từ đầu — bao gồm clone repo, cấu hình git worktrees cho multi-agent sprint, chạy validation scripts, và tạo PR đầu tiên.

<Callout kind="info">
Đọc [Getting Started FAQ](/help-center/faq/getting-started) trước nếu bạn chưa biết wiki này dùng để làm gì.
</Callout>

## Prerequisites

Trước khi bắt đầu, đảm bảo máy bạn có đủ:

| Requirement | Kiểm tra | Cài đặt nếu thiếu |
|-------------|----------|--------------------|
| macOS (Mac mini M4 hoặc tương đương) | `uname -m` → `arm64` | — |
| Git 2.40+ | `git --version` | `brew install git` |
| GitHub CLI | `gh --version` | `brew install gh` |
| Bash | Built-in trên macOS | — |
| Claude Code CLI | `claude --version` | Xem docs.anthropic.com |

### GitHub auth với workflow scope

```bash
# Kiểm tra auth status
gh auth status

# Nếu chưa login hoặc thiếu workflow scope:
gh auth login --scopes "repo,workflow"

# Verify sau khi login
gh auth status
# Token scopes phải bao gồm: repo, workflow
```

<Callout kind="warning">
Thiếu `workflow` scope khiến push bị reject với lỗi `refusing to allow a GitHub App to create or update workflow`. Xem [Install Error Troubleshooting](/help-center/troubleshooting/install-error) để fix.
</Callout>

<Steps>

## Git worktrees cho multi-agent sprint

Trong 4-agent sprint, mỗi worker agent chạy trong **git worktree riêng** để tránh conflict và cho phép làm việc song song.

<Steps>
  <Step title="Tạo worktrees cho từng agent" icon="copy" title-type="p">
    Chạy từ thư mục gốc của repo (sau khi clone):

    ```bash
    # Win 2 — agent-os-overview scope
    git worktree add ../documentationai-wiki-win2 win2-framework

    # Win 3 — workflows scope
    git worktree add ../documentationai-wiki-win3 win3-workflow

    # Win 4 — help-center và knowledge-qa scope
    git worktree add ../documentationai-wiki-win4 win4-reference-qa
    ```

    Mỗi lệnh tạo một thư mục mới ngang hàng với repo chính, checkout branch tương ứng.
  </Step>

  <Step title="Verify worktrees đã tạo thành công" icon="check" title-type="p">
    ```bash
    git worktree list
    ```

    Output mong đợi:
    ```
    /Users/phongto/Workspace/Claude/documentationai-wiki        341a758 [main]
    /Users/phongto/Workspace/Claude/documentationai-wiki-win2   abc1234 [win2-framework]
    /Users/phongto/Workspace/Claude/documentationai-wiki-win3   def5678 [win3-workflow]
    /Users/phongto/Workspace/Claude/documentationai-wiki-win4   ghi9012 [win4-reference-qa]
    ```
  </Step>

  <Step title="Mỗi agent làm việc trong worktree của mình" icon="user" title-type="p">
    ```bash
    # Win 4 làm việc tại:
    cd /Users/phongto/Workspace/Claude/documentationai-wiki-win4

    # Push từ worktree — đẩy lên branch win4-reference-qa
    git push origin win4-reference-qa
    ```

    **Quy tắc file ownership:** Mỗi agent chỉ edit files trong scope của mình. Xem [Multi-Agent Team FAQ](/knowledge-qa/multi-agent-faq) nếu cần cross-scope patch.
  </Step>
</Steps>

## Chạy validation scripts

Trước mỗi commit, **bắt buộc** chạy hai scripts:

### pre-commit-check.sh

Kiểm tra 4 lỗi MDX phổ biến:

```bash
# Chạy từ root của worktree
bash scripts/pre-commit-check.sh .
```

Script kiểm tra:
1. Unicode checkbox chars (`☐`, `□`) — phải dùng `- [ ]`
2. JSX tag collision (`<NUMBER` pattern) — phải viết "under 5%"
3. Missing frontmatter (`title:` và `description:` bắt buộc)
4. ASCII Vietnamese — phải dùng Unicode đầy đủ dấu

Xem chi tiết từng check tại [MDX Gotchas](/help-center/troubleshooting/mdx-gotchas).

### check-links.sh

Kiểm tra broken internal links:

```bash
bash scripts/check-links.sh .
```

Script verify rằng mọi đường dẫn `path` trong `documentation.json` có file `.mdx` tương ứng.

<Callout kind="info">
Nếu `check-links.sh` báo broken link cho file bạn vừa tạo, có thể file chưa được thêm vào `documentation.json`. Submit Cross-Scope Request cho Win 1. Xem [Install Error Troubleshooting](/help-center/troubleshooting/install-error#check-links-báo-broken) để biết thêm.
</Callout>

## Launch Claude Code CLI trong worktree

Mỗi agent chạy một Claude Code CLI session riêng trong worktree của mình:

```bash
# Mở terminal mới, cd vào worktree
cd /Users/phongto/Workspace/Claude/documentationai-wiki-win4

# Launch Claude Code

```

**Lưu ý:** Mỗi worktree có `.git` riêng (symlink về repo chính), nên `git status`, `git log`, v.v. đều hoạt động bình thường trong worktree.

## First commit và PR flow

<Steps>
  <Step title="Stage các file đã sửa" icon="plus-circle" title-type="p">
    ```bash
    # Stage file cụ thể (không dùng git add -A)
    git add help-center/faq/getting-started.mdx
    git add help-center/guides/setup-project.mdx
    ```
  </Step>

  <Step title="Commit với conventional format" icon="git-commit" title-type="p">
    ```bash
    git commit -m "feat(S2-W4-02): expand 3 help-center stubs to 100+ lines each"
    ```

    Conventional commit types: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`

    Task ID format: `S{sprint}-W{worktree}-{task_number}` (ví dụ: `S2-W4-02`)
  </Step>

  <Step title="Push lên worktree branch" icon="upload" title-type="p">
    ```bash
    git push origin win4-reference-qa
    ```

    Nếu là lần push đầu tiên:
    ```bash
    git push -u origin win4-reference-qa
    ```
  </Step>

  <Step title="Tạo PR trên GitHub" icon="git-pull-request" title-type="p">
    ```bash
    gh pr create \
      --title "feat(S2-W4-02): expand help-center stubs" \
      --body "$(cat <<'EOF'
    ## Summary
    - Expand 3 help-center stub files from 20 lines to 100+ lines each
    - getting-started.mdx: prerequisites, clone, deploy, first edit
    - setup-project.mdx: worktrees, validation, PR flow
    - install-error.mdx: auth, scope, pre-commit, MCP errors

    ## Test plan
    - [ ] bash scripts/pre-commit-check.sh . passes
    - [ ] bash scripts/check-links.sh . passes
    - [ ] All 3 files have valid frontmatter
    EOF
    )"
    ```
  </Step>

  <Step title="Cập nhật plans/progress.md" icon="file-text" title-type="p">
    Append vào `plans/progress.md`:
    ```
    [2026-04-20 14:28] [win4] Expanded 3 help-center stubs (S2-W4-02) — getting-started, setup-project, install-error
    ```
  </Step>
</Steps>

## Troubleshooting setup

Gặp lỗi khi setup? Xem [Install Error Troubleshooting](/help-center/troubleshooting/install-error) để fix các lỗi phổ biến:

- Git auth 401/403
- Push rejected vì thiếu workflow scope
- `pre-commit-check.sh` fails
- MCP không index sau deploy
