## Overview

Trang này tổng hợp các câu hỏi thường gặp nhất khi bắt đầu sử dụng **documentationai-wiki** và **Agent OS**. Đọc từ trên xuống nếu bạn mới bắt đầu, hoặc nhảy thẳng đến mục bạn cần.

<Callout kind="info">
Nếu bạn là agent trong 4-agent sprint (Win 2/3/4), hãy đọc thêm [Multi-Agent Team FAQ](/knowledge-qa/multi-agent-faq) sau khi đọc trang này.
</Callout>

## Wiki này là gì?

**documentationai-wiki** là knowledge base chính thức của hệ thống **Agent OS** — framework để điều phối nhiều Claude Code agent làm việc song song trên cùng một codebase. Wiki được deploy tự động qua **Documentation.AI** và phục vụ hai mục đích:

1. **Reference cho agents**: rules, workflows, scripts, troubleshooting guides
2. **Onboarding cho người dùng mới**: Phong To và các cộng tác viên sử dụng Claude Desktop Cowork để tạo marketing videos, automation, và Remotion exports

Wiki không phải là static site — nó được cập nhật liên tục trong mỗi sprint.

## Tôi cần gì để bắt đầu?

### Hardware & OS

- **Mac mini M4** (hoặc tương đương): macOS darwin 25.x trở lên
- Terminal với zsh hoặc bash

### Phần mềm bắt buộc

| Tool | Mục đích | Kiểm tra |
|------|----------|----------|
| Claude Desktop | Cowork mode — chạy Agent OS sessions | `claude --version` |
| Git 2.40+ | Clone repo, worktrees, commit | `git --version` |
| GitHub CLI (`gh`) | PRs, auth, workflow scope | `gh --version` |
| Bash | Chạy validation scripts | Built-in |

### GitHub config cần thiết

```bash
# Verify git identity
git config --global user.name
git config --global user.email

# Verify gh auth với workflow scope
gh auth status
# Phải thấy: ✓ Logged in to github.com as <username>
# Và: Token scopes: gist, read:org, repo, workflow
```

<Callout kind="warning">
Thiếu `workflow` scope là lỗi push thường gặp nhất. Xem [Install Error Troubleshooting](/help-center/troubleshooting/install-error) nếu push bị reject.
</Callout>

## Làm sao clone repo?

```bash
# Clone repo chính
git clone https://github.com/tominhphong/documentationai-wiki
cd documentationai-wiki

# Kiểm tra cấu trúc
ls -1
```

Bạn sẽ thấy các thư mục chính:

```
agent-os-overview/     # Tổng quan Agent OS — owned by Win 2
core-rules/            # CLAUDE.md rules, coding standards
workflows/             # Step-by-step workflow guides — owned by Win 3
help-center/           # FAQ, guides, troubleshooting — owned by Win 4
knowledge-qa/          # Multi-agent FAQ, QA patterns — owned by Win 4
scripts/               # Validation scripts (pre-commit, check-links)
plans/                 # Sprint plans, task board, progress log
documentation.json     # Navigation config — owned by Win 1
```

**Không cần `npm install` hay build step** — đây là content-only repo. Chỉ cần text editor và git.

## Wiki deploy ở đâu?

Documentation.AI tự động deploy khi có commit push lên nhánh `main`.

- **Live URL**: Được quản lý qua Documentation.AI dashboard (xem với Win 1 Lead)
- **Deploy time**: Khoảng 30–60 giây sau khi merge vào `main`
- **Auto-deploy trigger**: Push hoặc merge to `main` branch

Nếu bạn là worker agent (Win 2/3/4), bạn **không push thẳng lên main**. Bạn push lên worktree branch, tạo PR, Win 1 merge sau khi QA pass.

## Lần edit đầu tiên ra sao?

<Steps>
  <Step title="Chọn file cần sửa" icon="file-text" title-type="p">
    Tìm file `.mdx` tương ứng trong thư mục scope của bạn (ví dụ: `help-center/` nếu bạn là Win 4).
  </Step>

  <Step title="Edit trực tiếp" icon="edit" title-type="p">
    Mở file và chỉnh nội dung. Luôn giữ frontmatter hợp lệ với `title:` và `description:`.
  </Step>

  <Step title="Validate trước khi commit" icon="check-circle" title-type="p">
    ```bash
    bash scripts/pre-commit-check.sh .
    bash scripts/check-links.sh .
    ```
    Phải pass cả hai scripts. Xem [MDX Gotchas](/help-center/troubleshooting/mdx-gotchas) nếu có lỗi.
  </Step>

  <Step title="Commit với conventional format" icon="git-commit" title-type="p">
    ```bash
    git add help-center/path/to/file.mdx
    git commit -m "feat(S2-W4-XX): mô tả thay đổi ngắn gọn"
    ```
  </Step>

  <Step title="Push và tạo PR" icon="upload" title-type="p">
    ```bash
    git push origin win4-reference-qa
    gh pr create --title "feat: mô tả" --body "Summary of changes"
    ```
    Sau khi Win 1 merge, site live trong ~1 phút.
  </Step>
</Steps>
