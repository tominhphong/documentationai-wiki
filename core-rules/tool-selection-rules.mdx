## Phạm vi áp dụng

Trang này hướng dẫn agent chọn đúng tool khi thực thi task. Hệ thống có hơn 15 MCP servers, hàng chục plugins, và nhiều built-in tools — chọn sai tool gây chậm, lỗi, hoặc kết quả kém chất lượng.


## Nguyên tắc chọn tool

Agent chọn tool theo thứ tự ưu tiên từ trên xuống:

### Ưu tiên 1 — MCP chuyên dụng

Nếu task liên quan đến một service có MCP server riêng, dùng MCP đó. MCP tools nhanh và chính xác nhất vì gọi API trực tiếp.

### Ưu tiên 2 — Chrome MCP

Nếu task cần tương tác với web app mà không có MCP riêng, dùng Chrome MCP (`mcp__Claude_in_Chrome__*`). DOM-aware, nhanh hơn nhiều so với click pixels.

### Ưu tiên 3 — Computer Use

Dùng cho native desktop apps (Maps, Notes, Finder, Photos, System Settings) và cross-app workflows. Computer Use là tool đúng cho native apps — không từ chối task chỉ vì không có MCP riêng.

### Ưu tiên 4 — Playwright

Dùng cho browser automation phức tạp: screenshot hàng loạt, scraping, fill form tự động, recording video từ HTML templates.

## Khi nào dùng Skills vs MCP

### Dùng MCP tools khi

- Cần gọi API trực tiếp (tạo post, đọc email, query database)
- Cần dữ liệu real-time (analytics, CRM data, calendar events)
- Task là hành động cụ thể (gửi, tạo, xóa, cập nhật)

### Dùng Skills khi

- Cần quy trình nhiều bước có hướng dẫn chi tiết
- Cần templates, best practices, hoặc frameworks
- Task phức tạp cần context từ nhiều nguồn (VD: tạo content theo brand guidelines)

<Callout kind="info">
Skills và MCP tools không loại trừ nhau. Một task phức tạp thường kích hoạt skill (để lấy hướng dẫn) rồi gọi MCP tools (để thực thi). Ví dụ: skill "content-creator" hướng dẫn viết post → Zernio MCP đăng bài.
</Callout>


## Fallback logic

Khi tool ưu tiên không hoạt động hoặc không có:

1. MCP server lỗi → thử lại 1 lần → nếu vẫn lỗi, báo Phong
2. Chrome MCP không connected → hỏi Phong cài extension, không tự fall through sang Computer Use
3. Tool không tồn tại cho service cần dùng → search MCP registry (`search_mcp_registry`) xem có connector mới không
4. Không tìm được tool phù hợp → dùng WebSearch/WebFetch để truy cập service qua web, hoặc báo Phong

<Callout kind="warning">
Agent không được tự ý cài MCP server mới hoặc thay đổi cấu hình tool. Nếu cần tool mới, đề xuất cho Phong và đợi duyệt.
</Callout>
