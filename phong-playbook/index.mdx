## How Phong Uses Claude Code

Phong's setup follows a **Brain + Workers** pattern:

```
Brain (Claude Opus — orchestrator)
  ├── Agent 1 (Sonnet — research / content)
  ├── Agent 2 (Sonnet — code / MDX)
  ├── Agent 3 (Sonnet — validation / QA)
  └── Agent 4 (Sonnet — deployment / commit)
```

Key tools in daily use:

- **Claude Code (Cowork mode)** — primary interface, always-on
- **Desktop Commander MCP** — shell bridge between Claude and Mac mini M4
- **Zernio (Late) MCP** — social scheduling without leaving Claude
- **Remotion 4.0** — programmatic video generation
- **Vbee API** — Vietnamese text-to-speech
- **Git worktrees** — parallel agent branches with zero merge conflicts

The Brain (Opus model) holds the plan and delegates. Sonnet agents execute in parallel worktrees. Each agent owns distinct files — no overlap, no merge conflicts.

## Try This Yourself

### Prerequisites

1. Claude Code with Cowork mode enabled
2. A Mac (Desktop Commander MCP) or equivalent shell bridge
3. Git installed with worktree support (`git worktree add ...`)
4. A documentation platform (Mintlify, Docusaurus, or similar)

### Starting Template

```
# Brain prompt to kick off a new knowledge base sprint
"You are Brain (orchestrator). Spawn 3 parallel Sonnet agents via Task tool.
Each agent gets its own worktree. File ownership:
  Agent 1: content/topic-a/**
  Agent 2: content/topic-b/**
  Agent 3: scripts/validation/**
No agent edits another's files. Commit each worktree separately.
Merge order: Agent 3 (validation) → Agent 1 → Agent 2 → main."
```
